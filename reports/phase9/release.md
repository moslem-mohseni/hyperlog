## 3. The paper

`paper/main.tex` is an IEEE-TNSM-style manuscript with the full
structure of the submission:

- Abstract, Introduction, Related Work
- Method (architecture, three-stage training, calibration)
- Cross-system LOSO protocol
- Experimental setup (datasets, models, hardware, stats)
- Results: in-domain reproduction, cross-system LOSO, calibration,
  full ablation matrix
- Discussion and threats to validity
- Conclusion, reproducibility appendix pointer

Numerical cells that depend on the GPU run are marked **TBD**. They
auto-fill when the GPU artefacts land — `scripts/build_figures.py`
reads `reports/phase{2,4,5,6}/runs/...` and emits the figures; the
LaTeX `\input{...}` reads the same JSONs (when a future commit adds
the macros).

### Figures (5 PDFs)

1. `architecture.pdf` — schematic of the three-component cascade.
2. `loso_protocol.pdf` — flowchart for the zero-target-label protocol.
3. `reliability.pdf` — per-fold reliability curves (pre/post temperature scaling).
4. `loso_bars.pdf` — macro-F1 bars per held-out system.
5. `ablation_matrix.pdf` — Cliff's δ heat-map across the 8 ablation axes.

All five are regenerated by `scripts/build_figures.py` on every
commit. Missing JSON inputs yield a labelled "TBD" placeholder PDF
that LaTeX accepts gracefully.
